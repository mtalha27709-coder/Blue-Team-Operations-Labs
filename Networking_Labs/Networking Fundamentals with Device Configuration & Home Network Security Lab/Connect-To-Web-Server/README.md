# connect-to-web-server-packet-tracer
Simulation of client-to-web server connectivity using IP addressing in Cisco Packet Tracer, including ping verification and browser-based access.





🌐 Connect to a Web Server (Packet Tracer Project)
This project demonstrates how a client device connects to a web server using an IP address in a simulated network environment using Cisco Packet Tracer. It highlights how data packets travel across a network and how connectivity is established and verified.




🎯 Objective
To understand and analyze how network communication occurs between a client and a web server using IP addressing and basic networking protocols.





🛠️ Project Tasks
Verified connectivity using ICMP (ping command)
Sent packets from PC to Web Server (172.33.100.50)
Observed packet replies and TTL values
Connected to the web server using a web browser via IP address
Successfully loaded a webpage hosted on the server






📊 Results
✅ Successful ping replies received from the web server
⚠️ Initial packet loss observed due to ARP process (normal behavior)
🌐 Web page successfully accessed through browser






💡 Key Concepts Learned
IP Address-based communication
Packet flow in a network
Difference between DNS and direct IP access
ARP (Address Resolution Protocol) behavior
Client-server communication model





🛠️ Tools Used
Cisco Packet Tracer
Command Prompt (Ping utility)
Web Browser (HTTP access)




🚀 Outcome
This project strengthened my understanding of real-world networking concepts, particularly how clients communicate with servers using IP addresses. It also improved my ability to test, verify, and troubleshoot network connectivity.
