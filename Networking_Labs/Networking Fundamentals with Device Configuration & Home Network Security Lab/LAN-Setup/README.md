# 🖧 Packet Tracer - Create a LAN





## 📌 Overview
This project demonstrates how to design and configure a basic Local Area Network (LAN) using Cisco Packet Tracer.

The lab includes connecting network devices, configuring IPv4 addressing (DHCP & Static), and verifying connectivity between local and remote resources.

---




## 🎯 Objectives
- Connect network devices and end hosts
- Configure IPv4 addressing (DHCP & Static)
- Verify connectivity (LAN & Internet)
- Use networking commands (ipconfig, tracert)

---




## 🛠️ Network Setup




### Devices Used:
- Office Router
- Switch
- Admin PC (DHCP)
- Manager PC (DHCP)
- Printer (Static IP)
- Web Server (Internet)

---





## 🌐 IP Addressing


| Device        | IP Address        | Method  |
|--------------|------------------|--------|
| Admin PC     | DHCP             | Dynamic |
| Manager PC   | DHCP             | Dynamic |
| Printer      | 192.168.1.100    | Static  |
| Web Server   | 209.165.200.225  | Public  |

---





## 🔌 Connections
- Router ↔ ISP
- Router ↔ Switch
- PCs & Printer ↔ Switch

---





## ✅ Key Tasks Performed
- Connected devices using Ethernet cables
- Configured DHCP for PCs
- Assigned static IP to printer
- Verified connectivity using:
  - `ping`
  - `web browser`
- Used commands:
  - `ipconfig`
  - `tracert`

---





## 🔍 Key Learnings
- Difference between DHCP and Static IP
- Importance of default gateway
- DNS role (URL vs IP issue)
- Network troubleshooting basics

---






## ❗ Common Issue
If a website works with IP but not URL:
> DNS is not configured properly.
