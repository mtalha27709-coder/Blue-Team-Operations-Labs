# 🔐 Packet Tracer - Client Server Interaction



## 📌 Overview


This project demonstrates how a client (PC) interacts with a server in a network using **DNS** and **HTTP** protocols within Cisco Packet Tracer.

The simulation provides a clear understanding of how a web request is processed — from domain name resolution to web page delivery.



---



## 🎯 Objectives


- Observe client-server communication
- Understand DNS resolution process
- Analyze HTTP request and response flow
- Explore packet movement using simulation mode



---



## 🧠 Scenario

A simple network is configured where:
- A **PC acts as a client**
- A **Server provides DNS and HTTP services**



The client requests a web page (`www.example.com`), and the simulation tracks:
1. DNS query to resolve domain name into IP address
2. Server response with IP address
3. HTTP request sent to server
4. Web page delivered in segments
5. Acknowledgment from client



---



## ⚙️ Tools Used


- Cisco Packet Tracer
- DNS Protocol
- HTTP Protocol



---



## 🔍 Key Learning Points


- How DNS translates domain names into IP addresses
- How HTTP delivers web content
- Packet flow visualization using Simulation Mode
- OSI layer analysis through PDU details



---



## ▶️ How It Works


1. Switch to **Simulation Mode**
2. Apply filters for **DNS** and **HTTP**
3. Open PC → Web Browser → Enter URL
4. Run simulation to observe packet flow
5. Analyze events and PDUs layer by layer
